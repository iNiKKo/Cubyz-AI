## Explanation
Discussion on preventing player from placing blocks directly below them while airborne, with considerations for game balance and creative mode functionality. The issue revolves around a feature called 'nerd poling' that prevents players from placing blocks directly below them when they are airborne. When the player is airborne, they gain a hitbox under them that prevents blocks from being placed directly below them. Additionally, if methods are found to circumvent this, the player can be unable to place blocks if they're looking down at a certain angle. The discussion explores potential bypasses to this feature and suggests alternative solutions such as allowing enemies to build and destroy blocks, which could balance the game more effectively. Maintainers also discuss the idea of an accessory that allows creative mode functionality but without downsides like reduced damage with weapons. The accessory would allow players to place blocks in any orientation while airborne, similar to creative mode, but it would not have a downside such as reducing weapon damage. This feature is intended to be enabled only in Survival mode. The maintainer also mentions concerns about skybases and suggests designing around the fact that players can build whatever they want.
