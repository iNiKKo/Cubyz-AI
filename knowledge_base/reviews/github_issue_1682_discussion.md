## Explanation
Discussion about adding deadzones for movement and camera control to improve gameplay with controllers. The main points include implementing movement deadzones to allow natural input without requiring precise angles, and controller deadzones that restrict camera movement to eight directions to align with the blocky nature of the game world. Additionally, it is suggested that controller support should only activate when actual controller input is detected rather than just having a controller plugged in. The maintainer expresses difficulty playing Cubyz with a controller due to drifting issues and considers separating this feature into a new issue.
