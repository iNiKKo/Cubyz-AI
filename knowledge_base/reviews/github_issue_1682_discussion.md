## Summary
Discussion about adding deadzones for movement and camera control to improve player experience with controllers.

## Explanation
The discussion revolves around the need for implementing deadzones in Cubyz to enhance gameplay, particularly when using controllers. The main points include the addition of movement deadzones to allow for more natural input without requiring precise angles, and controller deadzones that restrict camera movement to eight directions to align with the blocky nature of the game world. There is a suggestion to activate controller support only when actual controller input is detected, rather than just having a controller plugged in. The maintainer expresses difficulty playing Cubyz with a controller due to drifting issues and considers separating this feature into a new issue.
