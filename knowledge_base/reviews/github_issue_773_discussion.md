## Summary
Discussion on mapping underlining and strike-through formatting to different characters, with a preference for using `__` for underline and `~~` for strikethrough due to their availability and familiarity.

## Explanation
The issue revolves around the use of underscores (`_`) in block and biome names causing unintended formatting. The discussion explores various options such as using asterisks (`*`) or different characters like tilde (`~`). However, there is a consensus on using double underscores (`__`) for underlining and double tildes (`~~`) for strikethrough due to their widespread availability and familiarity in markdown syntax. This decision aims to maintain consistency with common practices and ensure compatibility across different keyboard layouts.
