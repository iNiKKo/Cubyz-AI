## Summary
The Creative menu cannot be closed using the same 'C' input that opens it, unlike other menus. The discussion suggests adding a visual button inside the inventory to open the Creative menu and potentially removing or modifying the keyboard shortcut.

## Explanation
The issue revolves around inconsistent behavior in the Creative menu where pressing 'C' does not close the menu when already open, unlike other menus that use the same key for both opening and closing. For example, pressing E opens and closes the inventory with the same input. The steps to reproduce this issue are as follows: Pressing C opens the Creative menu but pressing C again while in the Creative menu does not close it; you must click away from the menu or press 'ESC' to leave it. Additionally, pressing C while already in the inventory should not close the menu.

The discussion explores potential solutions, including adding a visual button inside the inventory to access the Creative menu. This approach aims to improve user experience by making the functionality more intuitive and discoverable, especially for new players who may not be aware of the 'C' shortcut. The image provided shows an example of such a visual button.

There is also a suggestion to limit keyboard shortcuts to enhance controller support. At the end of the day, both a keybind and a visual button (with corresponding tooltip that mentions the keybind) would be good solutions.
