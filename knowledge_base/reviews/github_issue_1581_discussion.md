## Explanation
The `/sbb` command is introduced to allow players to save and manage structure building blocks (SBBs) along with their blueprints. It includes subcommands for saving, deleting, listing, loading, and placing SBBs. The exact syntaxes are as follows:

- `/sbb save <id> <--no-inline> <--world>` - Create an SBB and blueprint based on the clipboard. `--no-inline` can be used to generate an SBB zon file even if there was no child configuration set in the origin block. `--world` can be used to write to world assets instead of game assets.
- `/sbb delete <id>` - Deletes existing SBB.
- `/sbb list` - Lists IDs of available SBBs.
- `/sbb load <id>` - Loads blueprint associated with the SBB.
- `sbb place <id> <x> <y> <z> <--seed <seed>>` - Places an SBB in the world, resolving all its children. Optionally accepts x y z coordinates to place the origin block of the structure. The `--seed <seed>` parameter sets the seed used to sample child lists.

The `--world` option allows users to save SBBs directly into world-specific assets, which is preferred over game-wide assets for better modularity and organization within the game environment. Additionally, the `/sbb place` command differs from the `/paste` command in that it resolves all children of the structure as if generated by world generation, whereas `/paste` places blocks as-is without resolving any child configurations.
