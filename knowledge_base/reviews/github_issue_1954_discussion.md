## Summary
The discussion revolves around the use of tabs versus spaces for indentation in the Cubyz project, with maintainers defending the current formatting choices and users advocating for adherence to Zig's style guidelines.

## Explanation
The issue centers on whether the Cubyz project should adopt a code formatter that aligns with established Zig style guidelines or continue using its bespoke formatting rules. The maintainers argue that tabs improve code navigation, save bytes, and reduce asset loading times, while users prefer spaces for readability and consistency with industry standards. They also mention that editor extensions can help bridge the gap between different formatting preferences. A key concern is the impact on performance and disk space usage, although some points are debated as subjective or based on assumptions rather than empirical evidence.
