## Summary
Discussion on adding block data notation alongside block type, proposing integer notation, aliases for rotation definition, and Zon suffix similar to Minecraft.

## Explanation
The issue discusses the need to specify block data when using blocks with rotations. The proposer suggests three methods: integer notation (using a numeric value after the block ID), aliases in rotation definition (using known aliases for different data configurations), and using a Zon suffix similar to Minecraft's JSON format. The maintainer mentions that this feature is already implemented using another colon (`cubyz:oak_fence:3`) and is currently used only in the modern art biome.
