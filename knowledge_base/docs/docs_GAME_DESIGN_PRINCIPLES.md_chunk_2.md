## Summary
The remaining four "Break the Cycle" design rules: avoid clutter, use realism tastefully, avoid excessive additions, function over form.

## Explanation
**No Dimensions:** Instead of creating separate dimensions, these places can be physically integrated into Cubyz' massive world for the player to discover.

**No Teleportation:** Teleportation reduces immersion by diminishing exploration and preventing players from appreciating the scale of the world.

**No Automation:** Quick access to infinite resources discourages exploration, as players won't need to search for blocks they desire.

**Mobs Don't Respawn Naturally:** Clearing an area of monsters makes it safe for building and exploring. This rule also prevents mob farming.

**No Passive Animals:** Animals will either run from or defend against attacking players, requiring strategic hunting.

**No Unbreakable Tools:** Players won't want to make other tools if their amazing tool is unbreakable, leading to less use of low-tier materials in tool-making.

**Avoid Clutter:** Inventories can become cluttered with unwanted items. To mitigate this, find uses for existing items before adding new ones and prevent unnecessary items from entering the inventory.

**Use Realism Tastefully:** Cubyz is not a realistic game; it has whimsical rules inspired by the real world but never copied without a good reason.

**Avoid Excessive Additions:** Unless there's a genuine functional reason, avoid adding variants or new content. Prefer using or modifying existing content first.

**Function over Form:** When designing anything, prioritize its function to the overall gameplay before finalizing its design/theme. This ensures new features are meaningful.
